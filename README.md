GitHub's repository file viewer shows `index.html` as source code. Use the GitHub Pages URL above when you want the HTML pages to render in the browser.

## Structure

- `index.html`: searchable GitHub Pages homepage.
- `content/catalog.js`: the item registry that powers the homepage.
- `content/README.md`: catalog field reference.
- `items/`: default folder for future topic-based material.
- `assets/`: shared images, downloads, and supporting files.
- `docs/add-new-item.md`: checklist for adding new material.
- Root HTML files: existing shared pages with stable URLs.

## Add Something Good

1. Upload the new file to `items/topic-name/` or add a single HTML page at repo root.
2. Add one object to `content/catalog.js`.
3. Commit to `main`.

See [docs/add-new-item.md](docs/add-new-item.md) for the checklist.
