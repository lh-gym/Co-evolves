Add New Material

Keep the repo scalable by storing files predictably and registering every public item in one catalog.

Fast Path

  1. Put the new file in a stable location: root for short one-page URLs, or items/topic-name/ for new topic folders.
  2. Edit content/catalog.js.
  3. Copy an existing object, paste it at the end of the array, and update the fields.
  4. Commit to main.
  5. Open https://lh-gym.github.io/Co-evolves/ after GitHub Pages refreshes.

Naming Rules

  • Use lowercase folder names: items/sql-validation/.
  • Prefer hyphens over spaces: audit-evidence-checklist.html.
  • Keep public URLs stable once shared in Notion or a resume.
  • Use a private repo for private resumes, IDs, contact lists, or client-confidential material.
